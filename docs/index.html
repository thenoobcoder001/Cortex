Built-in terminal

A real shell, right next to chat.

Cortex ships a ConPTY-backed terminal pane that runs your shell with full PATH and ANSI rendering. Test the AI's edits without alt-tabbing.

  • Login shells on macOS and Linux so user-installed CLIs resolve
  • Augmented PATH for desktop-launcher environments on Windows
  • Clean ANSI handling — colors, cursor moves, and clears
  • Desktop-only by design — mobile relay can't reach the terminal

Live streaming

Every token, the moment it arrives.

All providers stream over Server-Sent Events. You see the model think in real time and can stop a run mid-generation. The chat lock releases the instant you interrupt.

  • SSE streaming for CLI providers (Claude, Codex, Gemini CLI)
  • Direct streaming for API providers (Gemini, Groq)
  • Stop button releases the lock immediately — no waiting
  • Recovered runs after a crash continue from the last token

Explain what `buildCleanEnv()` does and why it matters.
buildCleanEnv() builds the environment object passed to every spawned CLI child (claude, codex, gemini). It strips the host environment down to a small allowlist — PATH, HOME, proxy vars, and a handful of others — so secrets like GITHUB_TOKEN or AWS_* credentials living in your shell never leak into the AI subprocess.

Repo-aware by default

AI runs against real files on disk. Auto-written .claudeignore, AGENTS.md, and GEMINI.md keep it confined to the active repo.

Per-project memory

Chats save as JSON in .gpt-tui/chats/. Reopen the app and your conversations, model picks, and tool state are right where you left them.

Mobile relay

Cortex relay forwards requests from your phone to your desktop over an encrypted WebSocket. Rate-limited, audit-logged, paired per device.

Live diff viewer

A git status panel shows which files the AI changed, refreshing every 30 seconds in the background so you always see the latest delta.

Security-first

CLI children get a scrubbed env. API keys can't be overwritten remotely. Mobile callers blocked from terminal. Every relay call audit-logged.

Open in editor

One click opens the repo in VS Code, Cursor, Visual Studio, Windows Terminal, or File Explorer — auto-detected per OS with official brand icons.

Release notes

Every version of Cortex, with the detail you'd want from real release notes.

v6
v0.0.6 Latest Relay stability, light theme & error handling 2026-05-30

Relay stability

  • Auto-reconnect after the relay WebSocket drops post-auth — no more permanent disconnects from network blips.
  • In-flight chat streams are cleanly aborted when the relay drops instead of silently hanging.
  • Relay auth timeout reduced from 60s to 10s for faster failure detection.

Light theme fixes

  • Boot screen background, text, hex spinner, and progress bar are now fully themed (were hardcoded dark).
  • Linked devices and pairing-request rows no longer render a dark background in light theme.

Error handling

  • API rate-limit and quota errors (429, "usage limit reached", etc.) now surface as clear per-provider messages — e.g. "API limits reached for Claude."
  • Interrupt releases the chat lock immediately so a new chat starts without waiting.

Editor & settings

  • Fixed Antigravity editor launcher — command corrected from agy to antigravity.
  • curl unblocked in Claude's per-repo settings so network reachability checks work in chat sessions.

v5
v0.0.5 Codex model tiers, editor launcher & UI polish 2026-05-21

Codex model tiers

  • Updated Codex list to gpt-5.5, gpt-5.4, gpt-5.4-mini, gpt-5.3-codex (removed unsupported gpt-5.2-codex).
  • Each model now exposes four reasoning effort tiers: Extra High, High, Medium, Low.
  • Model picker is now a 3-level flyout — provider → model → effort tier.

Editor launcher

  • Added Visual Studio, File Explorer, and Windows Terminal to the open-in-editor menu.
  • All editor icons replaced with official brand SVGs.
  • Auto-detects OS — Windows Terminal / explorer on Windows, Terminal.app / open on macOS, x-terminal-emulator / xdg-open on Linux.

UI fixes

  • Fixed black screen on launch caused by a missing ModelPicker import and undefined handler refs.
  • Removed the model picker from the top navbar — model selection is now only in the composer.
  • Navbar spans the full width across sidebar and main content on all screen sizes.

Relay & linked devices

  • Linked devices stay visible in Settings regardless of relay connection state.
  • Approved devices no longer disappear after relay session expiry — approvals persist until explicitly removed.
  • Fixed stale polling that prevented the device list from refreshing after the initial load.

v4
v0.0.4 Desktop startup, relay & mobile workspace fixes 2026-05-19

Startup & desktop stability

  • Backend now runs as a child process instead of blocking the Electron main process.
  • Fixed "Startup interrupted" by using the actual bound backend port in the renderer URL.
  • Fixed startup hangs and "Not Responding" cases caused by repeated synchronous config/provider checks.
  • Removed the extra standalone backend from the dev command so Electron owns the backend lifecycle.

Cortex relay & mobile

  • Fixed re-pairing loops after relay reconnects by preserving signed relay sessions.
  • Mobile /api/status calls now use lite snapshots that skip expensive file trees and workspace diffs.
  • Mobile chat actions return lite snapshots — large repos like mobile-coapp open instantly.
  • Relay-proxied localhost requests are correctly treated as remote, not desktop-local.

Workspace & chat

  • Fixed chat rename behavior and persistence.
  • Faster project chat loading via lightweight /api/chats responses.
  • Cached CLI availability checks for Codex, Gemini CLI, and Claude CLI.
  • Auto-written .claudeignore files added for repo confinement and secret exclusion.

v3
v0.0.3 Cross-platform — macOS & Linux 2026-05-09

New features

  • macOS and Linux support — Cortex now runs on all three major desktop platforms.
  • Inline terminal panel runs directly inside the app.
  • Appearance / Theme selector moved to the top of Settings.
  • In-app update checks now report the right app version.

Critical fixes

  • Fixed "Not Responding" on Windows: AppConfigStore.load() was doing a synchronous disk read on every HTTP request inside the Electron main process. Replaced with an mtime-based in-memory cache.
  • Fixed relay crash on every launch — isRelaySessionExpired was referenced but never imported. Broken since v0.0.2.
  • Version strings now read from package.json instead of being hardcoded in the backend.

Polish

  • Login shells on Linux/macOS terminals so PATH includes user-installed CLIs.
  • Augmented PATH in buildCleanEnv for desktop-launcher environments.
  • Inline confirm/cancel UI for chat delete (replaced window.confirm).
  • Default theme is now dark; light-theme overrides for hardcoded text colors.
  • Relay security hardening and session expiry.

v0.0.2 Security & stability 2026-04-29

Security hardening

  • Relay callers can no longer overwrite API keys or sensitive config.
  • Device pairing guard — approvals required before any remote control of the desktop.
  • Rate limiting on every endpoint (30/min chat, 10/min config, 60/min default).
  • Relay audit log with 10 MB rotation.
  • CLI children no longer inherit host env vars (GitHub tokens, AWS keys, etc.).
  • Terminal access blocked for relay/mobile callers — desktop-only.
  • Auto-written .claudeignore and .claude/settings.json for per-repo AI confinement.

Fixes

  • Fixed Windows spawn crash for Node.js installed in paths with spaces.
  • Fixed CORS blocking the Vite dev renderer on 127.0.0.1:5173.
  • Fixed spawn cmd.exe ENOENT caused by missing PATH in clean env.
